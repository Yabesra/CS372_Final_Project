# Attribution

## AI‑Generated Code (per course policy)
- The Streamlit UI structure in `src/app.py` (layout, file uploader, buttons, progress bars) was generated with AI assistance (ChatGPT, March 2026) and then adapted.
- The PDF text extraction function `extract_text_from_pdf()` was AI‑generated and adapted.
- The heuristic risk scoring functions (`compute_risk_score`, `fairness_score_from_risk`, `risk_level`, `compute_context_coverage`) were written with AI assistance.

## Code Reused from Course Assignments
- `src/retriever.py` – directly adapted from the RAG assignment (mean pooling embedding, vectorized cosine similarity).
- `src/evaluate.py` – `evaluate_answer()` function copied from the RAG assignment; evaluation metrics (accuracy, precision, recall, confusion matrix) from the Evaluation assignment.
- Instruction prompt format (`Context: ...\nInstruct: ...\nOutput:`) from the LLM instruction‑tuning assignment.

## External Libraries
- `streamlit` – web UI framework.
- `transformers` – Hugging Face library for loading `distilgpt2` and tokenizers.
- `torch` – PyTorch backend for model inference.
- `sentence-transformers` – used for `all-MiniLM-L6-v2` embeddings (via `retriever.py`).
- `scikit-learn` – metrics (accuracy, precision, recall, confusion matrix).
- `pypdf` – PDF text extraction.
- `numpy` – vector operations for embeddings.

## Models
- `distilgpt2` – lightweight causal language model for generating risk summaries (Hugging Face).
- `sentence-transformers/all-MiniLM-L6-v2` – embedding model for dense retrieval (Hugging Face).

## Datasets
- Gold test set for evaluation – AI generated by DeepSeek (10 clauses, 5 risky / 5 safe). Not sourced from external data.
- No external fine‑tuning datasets, but CUAD's labeling was used as a reference for the clause type taxonomy and to guide the design of the weighted risk assessment.
